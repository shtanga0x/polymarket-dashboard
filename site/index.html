How this dashboard works

What you're looking at

This dashboard tracks a curated list of consistently successful Polymarket traders and aggregates their portfolios into one combined view. The idea: when many independently profitable traders converge on the same market outcome, that consensus is signal. Two sister dashboards share this exact codebase: core.shtanga.xyz — a small, strictly curated tier-1 list — and watch.shtanga.xyz — a broader watchlist (tier 1 + tier 2 candidates under evaluation).

Sections

  • Traders — the tracked wallets: label, portfolio value, all-time PnL, USDC balance and open-position count. Click a trader to open their Polymarket profile. Use the search box to filter.
  • Portfolio — the heart of the dashboard. Every market outcome held by tracked traders, aggregated: how many traders hold it, combined exposure ($ and shares), average entry price, % allocation, and chained momentum columns (1h / 24h / 1w Change). Sort by any column; hover the change columns for a per-trader breakdown tooltip.
  • Recent Changes — the live feed of position changes: entries, exits, adds and trims, newest first. The eye icon in the Portfolio's last column filters this feed to one market — that's the fastest way to answer "who just moved here?".
  • Checker — paste any wallet address to inspect its current positions, exposure and all-time PnL. Works for wallets outside the tracked list too — use it to vet a trader before suggesting them for the watchlist.

What to pay attention to

  • Trader count beats raw exposure. $500k from one whale is one opinion; $50k from nine independent traders is consensus. Sort the Portfolio by Traders to surface it. The Traders column also shows the last 24h of churn — e.g. 6 (+2/−1) means 6 holders now, 2 entered and 1 exited within 24h. Fresh entries are the strongest version of the consensus signal.
  • Momentum columns are chained, non-overlapping windows: 1h Change covers the last hour, 24h Change the day before it (1h–24h ago), 1w Change the week before that (1–7 days ago) — each trade is counted in exactly one column, so you can read the evolution window by window (e.g. accumulated last week, quiet yesterday, buying again this hour). Fresh money in the 1h column on a previously quiet market is the classic early signal.
  • Balanced pairs filter — some traders hold both YES and NO of the same market (market-making / hedging, not directional opinion). The filter hides those so you see real conviction only.
  • Expiration — a heavy position expiring in days is a sharper signal than the same position six months out.
  • Freshness — the timestamp next to the tabs shows the last data refresh. Data updates every ~2 minutes; the page picks it up automatically (auto-reload pauses after you've been inactive, the ↻ button forces a refresh).

Architecture (for the curious)

  • Pipeline — a GitHub Actions job runs every minute: it pulls each tracked wallet's positions, balances and activity from the Polymarket APIs (through rotating proxies, with retries), scrapes all-time PnL (cached ~1h), aggregates everything, and diffs against the previous run to produce the Recent Changes feed.
  • Storage — results are published to Cloudflare R2 as immutable snapshots plus a tiny metadata.json pointer that flips only after a snapshot is fully uploaded — so the page can never read a half-written update.
  • Frontend — this static page polls only the pointer (~1 KB) and re-downloads the data files just when the snapshot actually changed.
  • Notifications — a Telegram bot watches the same data feed and posts significant position changes (entries, exits, milestones) with its own snapshot state, so it degrades gracefully if a refresh is delayed.
  • One codebase — core and watch differ only in a config file and a traders CSV; every feature and fix ships to both sites simultaneously.

Caveats

  • This is a research tool, not an execution feed. Data is delayed by up to a few minutes; by the time you see a move, prices may have already adjusted.
  • Survivorship and selection bias. Traders are added because they performed well in the past — past performance does not guarantee they keep performing. The list is curated by a human and can simply be wrong.
  • Consensus is signal, not certainty. Tracked traders can be wrong together (shared narratives, copycat flows), early, or hedged on venues this dashboard can't see.
  • Position ≠ opinion. A position can be a hedge, market-making inventory, an exit in progress, or a leg of a structure. The balanced-pairs filter catches the obvious cases, not all of them.
  • Numbers are best-effort. PnL comes from Polymarket's own profile stats and can lag up to an hour; exposure uses current market prices, which move; API hiccups can briefly under-count a trader.
  • Thin markets distort. In low-liquidity markets a single tracked trader can be most of the volume — "3 traders, $80k" means little if the whole book is $100k.
